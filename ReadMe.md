# CG2評価課題

1. 2つの三角形の描画 (Scene 1)
画面上に独立した2つの三角形を配置し、プログラム内で transform（1つ目）と transform2（2つ目）に分けて、位置・回転・スケール（大きさ）を管理する構造体がそれぞれ用意されています。

GPUに対して1つ目の色・座標・テクスチャをバインドして描画、次に2つ目の色・座標・テクスチャをバインドして描画という手順を連続して行うことで、2つの異なる三角形を画面に表示しています。

開発用UIを使って、それぞれの三角形の位置を動かしたり、テクスチャ（画像）や色をリアルタイムに変更できるようになっています。

実行画面で一つの三角形をいじるともう一個の三角形は影響に与えることがないようにしている。

2. 大量の三角形による演出 (Scene 2)
最大300個（kMaxParticles = 300）の三角形を、弾幕のように配置してダイナミックに動かすサイバー風ワープトンネルのエフェクト演出です。

演出を「かっこよく」見せるためにすべての三角形がカメラに向かって奥から手前へ高速に移動します。
サイン・コサイン波（sinf, cosf）を使い、手前に迫るほど円状に大きく広がりながらうねる軌道を作っています。
1つのパーティクルにつき、三角形を45度ずらして2枚交差（立体的なクロス形状）させて描画することで、ペラペラ感を減らしています。

カメラを通り過ぎる手前では自然に消えるように、透明度（Alpha値）を滑らかに変化させています。
